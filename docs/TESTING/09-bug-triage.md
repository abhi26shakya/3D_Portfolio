# Bug Triage

Book X — Testing

## Overview

Bug triage defines the process of identifying, categorizing, prioritizing, and managing issues discovered during testing of the 3D Portfolio. It ensures that defects are handled efficiently and systematically.

## Objectives

- Identify and classify bugs
- Prioritize fixes based on severity
- Improve development workflow
- Maintain product stability
- Ensure clear communication between teams

## Bug Classification

### 1. Severity Levels
- Critical: System crash or broken core functionality
- High: Major feature not working correctly
- Medium: Partial feature issues or inconsistencies
- Low: Minor UI or cosmetic issues

### 2. Priority Levels
- P0: Must fix immediately
- P1: Fix in current release
- P2: Fix in next release
- P3: Low priority / backlog

## Triage Workflow

1. Bug reported via testing or user feedback
2. Reproduce the issue
3. Validate severity and priority
4. Assign to developer
5. Track progress
6. Verify fix
7. Close issue

## Tools

- GitHub Issues
- Project boards (Kanban)
- CI error logs
- Browser developer tools

## Best Practices

- Reproduce bugs before logging
- Provide clear steps to reproduce
- Attach screenshots or logs
- Avoid duplicate bug reports
- Regularly review open issues
