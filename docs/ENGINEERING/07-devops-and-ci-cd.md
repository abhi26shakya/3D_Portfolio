# DevOps and CI/CD

Book V — Engineering

## Overview

DevOps combines software development and IT operations to enable faster, more reliable software delivery. Continuous Integration (CI) and Continuous Deployment (CD) automate testing, building, and deployment.

## DevOps Principles

- Collaboration between development and operations
- Infrastructure as Code (IaC)
- Automation-first mindset
- Continuous monitoring and feedback

## CI Pipeline

1. Commit code
2. Run automated tests
3. Build artifacts
4. Perform static analysis
5. Package application

## CD Pipeline

- Deploy to staging
- Run integration tests
- Perform approval checks
- Deploy to production
- Monitor rollout

## Common Tools

- GitHub Actions
- Docker
- Kubernetes
- Terraform
- Jenkins
- Prometheus
- Grafana

## Best Practices

- Automate repetitive tasks.
- Keep deployments small and frequent.
- Monitor production continuously.
- Roll back safely when failures occur.
