# Design Patterns

Book V — Engineering

## Overview

Design patterns are reusable solutions to recurring software design problems. They provide proven approaches for organizing code while improving maintainability and flexibility.

## Creational Patterns
- Singleton
- Factory Method
- Abstract Factory
- Builder
- Prototype

## Structural Patterns
- Adapter
- Bridge
- Composite
- Decorator
- Facade
- Flyweight
- Proxy

## Behavioral Patterns
- Observer
- Strategy
- Command
- State
- Iterator
- Mediator
- Template Method
- Chain of Responsibility

## Best Practices
- Prefer composition over inheritance.
- Use patterns only when they simplify the design.
- Avoid over-engineering by introducing unnecessary abstractions.
