# Security Engineering

Book V — Engineering

## Overview

Security engineering integrates security into every stage of the software development lifecycle. Rather than treating security as an afterthought, modern systems adopt a security-by-design approach.

## Core Principles

- Confidentiality
- Integrity
- Availability (CIA Triad)
- Least Privilege
- Defense in Depth
- Zero Trust

## Common Security Practices

- Authentication and Authorization
- Encryption in transit and at rest
- Secret management
- Input validation and output encoding
- Dependency and vulnerability scanning
- Secure coding standards

## Security Testing

- Static Application Security Testing (SAST)
- Dynamic Application Security Testing (DAST)
- Penetration Testing
- Threat Modeling

## Best Practices

- Keep software dependencies updated.
- Enable logging and auditing.
- Regularly rotate secrets and credentials.
- Perform periodic security reviews.
- Automate security checks within CI/CD pipelines.

Security is a continuous engineering discipline that protects users, data, and infrastructure throughout the system lifecycle.
