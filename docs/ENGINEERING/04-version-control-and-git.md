# Version Control and Git

Book V — Engineering

## Overview

Version control systems track changes to source code, documentation, and assets over time. Git is the industry-standard distributed version control system that enables collaboration, experimentation, and reliable history management.

## Why Version Control?

- Preserve project history
- Enable collaboration
- Recover previous versions
- Support branching and experimentation
- Simplify code reviews and releases

## Core Git Concepts

- Repository
- Commit
- Branch
- Merge
- Remote
- Pull Request

## Common Workflow

1. Create a feature branch.
2. Make small, focused commits.
3. Push changes to the remote repository.
4. Open a pull request.
5. Review, merge, and delete the branch.

## Best Practices

- Write meaningful commit messages.
- Keep commits atomic.
- Rebase or merge regularly.
- Never commit secrets or credentials.
- Use `.gitignore` effectively.
