# AI Behavior

Book VIII — Character

## Overview

AI behavior defines how non-player characters (NPCs) and interactive agents respond to user actions and environmental events. A well-designed behavior system creates believable interactions while keeping characters predictable and maintainable.

## Behavior Types

- Idle
- Patrol
- Follow
- Guide
- Interact
- React to user actions

## Decision Systems

- Finite State Machines (FSM)
- Behavior Trees
- Utility-based AI
- Event-driven behaviors

## Interaction Triggers

- Player proximity
- User input
- Time-based events
- Story progression
- Environmental conditions

## Best Practices

- Keep behaviors modular.
- Prioritize deterministic interactions.
- Minimize unnecessary processing.
- Test edge cases and transitions.
- Document all states and triggers.

A structured AI behavior system makes characters feel responsive, believable, and engaging while remaining scalable for future expansion.
