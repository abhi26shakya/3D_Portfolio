# Dialogue System

Book VIII — Character

## Overview

The dialogue system defines how characters communicate with users throughout the 3D Portfolio. It combines text, timing, interaction logic, and presentation to create natural and engaging conversations while guiding users through the experience.

## Dialogue Types

- Greetings
- Tutorials
- Contextual hints
- Project explanations
- Story conversations
- Farewell messages

## Components

- Dialogue manager
- Conversation nodes
- Response options
- Trigger conditions
- UI presentation
- Voice and subtitles

## Interaction Flow

1. Trigger conversation
2. Display dialogue
3. Wait for user response
4. Process selected option
5. Advance or end conversation

## Best Practices

- Keep dialogue concise.
- Match the character's personality.
- Avoid unnecessary repetition.
- Support accessibility with subtitles.
- Design conversations to be skippable and replayable.

A well-structured dialogue system strengthens storytelling, improves user guidance, and creates memorable interactions within the portfolio world.
