# NPC Design

Book VIII — Character

## Overview

Non-Player Characters (NPCs) enrich the 3D Portfolio by guiding visitors, presenting information, and making the virtual environment feel alive. Each NPC should have a clear purpose, recognizable appearance, and meaningful interactions.

## NPC Roles

- Guide
- Receptionist
- Mentor
- Project Expert
- Archivist
- Ambient Citizen

## Design Principles

- Distinct visual identity
- Clear functional role
- Simple interaction patterns
- Consistent behavior
- Accessible dialogue

## Interaction Features

- Context-aware conversations
- Directional guidance
- Project explanations
- Achievement highlights
- Environmental storytelling

## Technical Considerations

- Modular behavior components
- Shared animation library
- Optimized models and textures
- Event-driven interactions
- State persistence

## Best Practices

- Give every NPC a purpose.
- Avoid repetitive dialogue.
- Keep interactions concise and helpful.
- Maintain visual consistency.
- Test NPC behavior across different scenarios.

Well-designed NPCs transform a static portfolio into an engaging, interactive experience that encourages exploration and discovery.
